# Test Cases Sprint 1 — Главная страница со списком записей и фильтром по статусу

## Setup

- Запустить локально: `npm run dev`.
- Открыть `http://localhost:5173`.
- Для подготовки данных: перед тестами `T-1`–`T-5` положить в `localStorage` ключ `records` со значением минимум 3 записей (1 `active`, 1 `done`, 1 `active`).
- Для теста `T-6` — очистить `localStorage`.
- Браузер для прохождения: Chrome (основной). Регрессионный прогон в Safari после фикса бага.

## Test cases

| # | Test Case | Expected | Status |
|---|-----------|----------|--------|
| T-1 | Открыть главную с заполненным localStorage | Виден заголовок страницы, фильтр All активен, в списке все 3 записи в порядке от новых к старым | ok |
| T-2 | Кликнуть фильтр Active | В списке остаются только записи со статусом `active` (2 шт). Кнопка Active подсвечена, All и Done — нет | ok |
| T-3 | Кликнуть фильтр Done | В списке остаётся 1 запись со статусом `done`. Кнопка Done подсвечена | ok |
| T-4 | Кликнуть фильтр All после Done | Возвращаются все 3 записи, кнопка All подсвечена | ok |
| T-5 | Нажать Tab с фокусом на странице, проверить что фокус последовательно проходит по 3 кнопкам фильтра, потом по записям | Tab-навигация работает, focus visible | bug |
| T-6 | Открыть главную с пустым localStorage | Виден empty state с текстом из `copy.emptyList`. Кнопки фильтра видны, но переключение не приводит к ошибкам | ok |
| T-7 | Перезагрузить страницу (F5) после переключения фильтра на Active | После перезагрузки фильтр сбрасывается на All (фильтр не персистится — это ожидаемо, см. ADR-001) | ok |
| T-8 | Регрессия: открыть `/settings` и `/about`, проверить что они работают как раньше | Страницы открываются, ничего не сломано | ok |
| T-9 | Прогнать визуально в Safari после фикса B-1 | Кнопки фильтра, список и empty state выглядят корректно | ok |

Статусы: `⬜` (не проверен), `ok` (пройден), `bug` (баг, описать в таблице ниже).

## Bugs found

| ID | Severity | Test case # | Description | Steps to reproduce |
|----|----------|-------------|-------------|-------------------|
| BUG-1 | Medium | T-5 | При Tab-навигации фокус «проваливается» через кнопки фильтра без видимой подсветки. Активная кнопка фильтра вообще не получает focus-ring, ContrastChecker показывает что focus visible — нет | 1. Открыть главную. 2. Нажать Tab. 3. Видно, что фокус ушёл куда-то на страницу, но визуально не понятно куда — нет outline на кнопках |

## Decision after QA

- BUG-1 → severity Medium, но связан с темой C-2 из code review (a11y кнопок). Пользователь решил: вместо точечного фикса в Sprint 1, BUG-1 переезжает в a11y-спринт (после Sprint 3), где будут разом добавлены `aria-pressed`, focus-ring, нормальная keyboard navigation. Sprint 1 закрывается с пометкой Partial, BUG-1 трекается в `docs/project/JTBD.md` секция Deferred.
